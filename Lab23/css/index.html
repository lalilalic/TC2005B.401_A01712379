Lab 23 - Store Procedures

Una desventaja es que son difíciles de revisar cuando tienen errores, porque no puedes debuggearlos tan fácil como el código normal de tu aplicación. Otra desventaja es que si necesitas cambiar algo, tienes que entrar directo a la base de datos a modificarlo, y eso hace difícil llevar un control de versiones con git. Por último, están amarrados al gestor de base de datos que uses, o sea si usas MySQL y después quieres cambiar a otro sistema, tienes que reescribir todos tus procedimientos desde cero.
